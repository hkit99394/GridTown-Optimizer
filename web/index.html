              Choose an optimizer

              Recommended quality path: use Auto when overall answer quality matters most. Auto starts with a capped fast Greedy seed, then LNS and bounded CP-SAT spend the deeper improvement budget. Standalone Greedy stays available as the heavy heuristic / advanced inspection mode.

              Presets keep Greedy, LNS, and bounded CP-SAT one click away. Auto remains the recommended quality path.

              Auto is the recommended quality path. It uses a capped fast Greedy seed stage, then LNS repairs neighborhoods and bounded CP-SAT polishes the incumbent. Auto generates per-stage random seeds; standalone Greedy and CP-SAT seed fields are disabled in this mode.
              Greedy is the heavy standalone heuristic / advanced inspection mode. Use Auto when you want the fast seed stage plus LNS and bounded CP-SAT follow-on improvement.
              LNS is the manual improvement mode. It starts from a greedy or displayed seed, then repairs one neighborhood at a time with CP-SAT. It still needs the Python OR-Tools backend.

              No displayed output is available to use as an LNS seed.

              CP-SAT uses the Python OR-Tools backend. If the environment is not set up yet, use npm run setup:cp-sat first. Leave the time limit blank to let it run until it finishes or you stop it manually.
              Portfolio CP-SAT launches up to four exact paths inside a bounded per-worker budget. Internal worker lanes are capped at eight total.

              No displayed output is available to use as a CP-SAT hint.
